# 웨딩 비용 정리 블로그 개발 로드맵

Notion을 CMS로 활용하여 웨딩 비용 경험을 카테고리별로 정리하고 자동 발행하는 개인 블로그를 구축합니다.

## 프로젝트 개요

웨딩 비용 정리 블로그는 결혼을 준비 중이거나 완료한 개인 및 커플을 위한 콘텐츠 공유 플랫폼으로, Notion 데이터베이스에 작성한 글을 자동으로 블로그에 발행합니다.

- **목표**: 웨딩 비용 정리 블로그 MVP(v1.0) 완성 및 배포
- **범위**: MVP 핵심/지원 기능 구현 (F001 ~ F012)
- **예상 기간**: 약 4주 (1인 개발자 기준)
- **릴리스 버전**: v1.0
- **배포 플랫폼**: Vercel

### 핵심 기능

- **Notion API 연동 (F001)**: Notion 데이터베이스에서 발행된 글을 자동 조회하는 CMS 기반
- **글 목록/상세 (F002, F003)**: 카드 형태의 글 목록과 Notion 블록 기반 본문 렌더링
- **카테고리 필터링 (F004)**: 예식장, 스드메, 예물 등 카테고리별 글 탐색
- **검색 (F005)**: 제목 및 태그 기반 글 검색
- **반응형 + SEO (F006, F010)**: 모든 기기 최적화 및 검색 엔진/소셜 공유 대응

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - `/tasks` 디렉토리에 새 작업 파일 생성
   - 명명 형식: `XXX-description.md` (예: `001-notion-client.md`)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)
   - 예시를 위해 `/tasks` 디렉토리의 마지막 완료된 작업 참조
   - 새 작업의 경우 빈 박스로 시작하며 변경 사항 요약은 비워둠. 초기 상태 샘플로 `000-sample.md` 참조

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 구현 완료 후 Playwright MCP를 사용한 E2E 테스트 실행
   - 테스트 통과 확인 후 다음 단계로 진행
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 ✅로 표시

## 주요 마일스톤

- **M1: Notion API 연동 완료** (Phase 1 종료) — Notion 데이터 조회 및 파싱 검증
- **M2: 모든 페이지 UI 완성** (Phase 2 종료) — 더미 데이터 기반 전체 화면 구현
- **M3: 핵심 기능 구현 완료** (Phase 3 종료) — 실제 데이터 연동 및 검색/필터링 동작
- **M4: 배포 준비 완료** (Phase 4 종료) — Vercel 배포 및 v1.0 릴리스
