# 웨딩 비용 정리 블로그 v2.0 개발 로드맵

공개 블로그에 더해 **운영자 전용 관리자 대시보드**를 구축하여, 카테고리·글을 한눈에 관리하고 클라이언트에게 공유 링크를 손쉽게 전달할 수 있는 콘텐츠 운영 환경을 완성합니다.

## 프로젝트 개요 (v2.0 비전)

MVP(v1.0)에서는 Notion CMS 기반의 공개 웨딩 비용 블로그(글 목록/상세, 카테고리 필터링, 검색, 반응형, SEO, Vercel 배포)를 완성했습니다.
v2.0은 **블로그 운영자(1인)를 위한 관리 도구**를 추가하여 콘텐츠 운영 효율을 높이는 것을 목표로 합니다. 공개 페이지는 그대로 유지한 채, `/admin` 경로 아래에 인증으로 보호되는 관리자 영역을 신설합니다.

- **목표**: 관리자 대시보드 + 운영 편의 기능을 통한 콘텐츠 관리 효율화
- **범위**: 관리자 인증/레이아웃, 카테고리·글 조회 대시보드, 클라이언트 공유 링크 복사, 글 상태 관리, 통계, 다크 모드 완성, 댓글/피드백 관리 (F101 ~ F108)
- **예상 기간**: 약 5~6주 (1인 개발자 기준)
- **릴리스 버전**: v2.0 (단계별 v2.0-beta → v2.0 → v2.1)
- **배포 플랫폼**: Vercel (기존 환경 재사용)
- **개발 방법론**: 구조 우선 접근법(Structure-First Approach)

### v2.0 핵심 기능

- **관리자 인증/레이아웃 (F101)**: `/admin` 보호 라우트, 관리자 전용 사이드바 레이아웃
- **카테고리 관리 대시보드 (F102)**: 카테고리 목록 조회, 카테고리별 글 개수·금액·상태 표시 (조회 전용, 생성/수정/삭제는 v2.0 제외)
- **카테고리별 글 관리 (F103)**: 선택 카테고리의 글 목록 관리 뷰
- **클라이언트 공유 링크 복사 (F104)**: 카테고리별 공개 URL을 클릭 한 번으로 복사
- **글 상태 관리 (F105)**: 발행/비발행/아카이브 상태 조회 및 표시
- **간단한 통계 (F106)**: 조회수, 카테고리별 글 개수 등 운영 지표
- **다크 모드 완성 (F107)**: 기존 스캐폴딩된 테마 시스템을 전 페이지에 안정화 적용
- **사용자 피드백/댓글 관리 (F108)**: 방문자 피드백/댓글 수집 및 관리자 검토

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스(`src/app`, `src/lib/notion`, `src/components`)를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - `/tasks` 디렉토리에 새 작업 파일 생성
   - 명명 형식: `XXX-description.md` (v1.0이 `001`~`014`를 사용했으므로 v2.0은 `101-admin-auth.md`부터 시작)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - **API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)**
   - 예시를 위해 `/tasks` 디렉토리의 마지막 완료된 작업 참조 (예: 현재 `103` 작업이면 `102`, `101` 참조)
   - 완료된 작업은 체크된 박스와 변경 사항 요약을 반영. 새 작업은 빈 박스로 시작하며 변경 사항 요약은 비워둠. 초기 상태 샘플로 `000-sample.md` 참조

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - **API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수**
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 구현 완료 후 Playwright MCP를 사용한 E2E 테스트 실행
   - 테스트 통과 확인 후 다음 단계로 진행
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 ✅로 표시

## 주요 마일스톤

- **M5: 관리자 영역 골격 완성** (Phase 1 종료) — `/admin` 인증 보호 + 관리자 레이아웃 골격, 타입 정의 완료
- **M6: 관리자 대시보드 UI 완성** (Phase 2 종료) — 더미 데이터 기반 관리자 전 화면 구현 (v2.0-beta)
- **M7: 관리자 핵심 기능 구현 완료** (Phase 3 종료) — 실제 데이터 연동, 카테고리 조회, 공유 링크 복사 동작 (v2.0)
- **M8: 고도화 및 최적화 완료** (Phase 4 종료) — 글 상태/통계/다크모드/피드백 + 성능 검증 (v2.1)
