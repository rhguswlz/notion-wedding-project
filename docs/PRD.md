# 웨딩 비용 정리 블로그 MVP PRD

## 🎯 핵심 정보

**목적**: Notion을 CMS로 활용하여 웨딩 비용을 카테고리별로 정리하고 개인 블로그에서 자동으로 발행하여 경험을 공유하기

**사용자**: 결혼 준비 중이거나 완료한 개인 및 커플이 본인의 기록을 남기고 경험을 공유하려는 사람

---

## 🚶 사용자 여정

```
1. 홈 페이지 (첫 방문)
   ├─ 최근 글 목록 확인
   └─ 카테고리 네비게이션 확인

2. 사용자 선택 분기점
   ├─ [카테고리별로 보기] → 카테고리 페이지
   ├─ [검색하기] → 글 목록 (검색 결과)
   └─ [글 제목 클릭] → 글 상세 페이지

3. 카테고리 페이지 / 검색 결과 페이지
   ├─ 필터링된 글 목록 표시
   ├─ 글 카드 클릭
   └─ → 글 상세 페이지로 이동

4. 글 상세 페이지
   ├─ Notion 블록 기반 본문 렌더링
   ├─ 카테고리/태그 정보 표시
   └─ [다른 글 보기] → 홈 또는 카테고리 페이지로 이동

5. 완료 / 다른 탐색
   ├─ 홈으로 돌아가기
   ├─ 다른 카테고리 탐색
   └─ 다른 글 읽기
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | Notion API 연동 | Notion 데이터베이스에서 발행된 글 자동 조회 | 서비스의 핵심 기반 (CMS) | 홈 페이지, 카테고리 페이지 |
| **F002** | 글 목록 표시 | 최근 글을 카드 형태로 게시글 목록 표시 | 사용자가 가장 먼저 마주하는 핵심 기능 | 홈 페이지 |
| **F003** | 글 상세 보기 | Notion 블록을 기반으로 글의 전체 본문 렌더링 | 글 내용 확인의 필수 기능 | 글 상세 페이지 |
| **F004** | 카테고리 필터링 | 선택한 카테고리의 글 목록만 조회 및 표시 | 사용자의 관심 카테고리별 탐색 지원 | 홈 페이지, 카테고리 페이지 |
| **F005** | 검색 기능 | 제목 및 태그로 글 검색 | 특정 글을 빠르게 찾는 사용자 니즈 | 홈 페이지 |
| **F006** | 반응형 디자인 | 모바일/태블릿/데스크톱 최적화된 레이아웃 | 모든 기기에서 최적의 사용자 경험 제공 | 모든 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 기본 SEO 설정 | 페이지별 메타 데이터 및 OG 태그 설정 | 검색 엔진 최적화 및 소셜 공유 기본 구성 | 모든 페이지 |
| **F011** | 글 메타데이터 표시 | 발행일, 카테고리, 태그 정보 표시 | 글 정보 확인을 위한 필수 요소 | 글 상세 페이지 |
| **F012** | 카테고리 네비게이션 | 전체 카테고리 목록 네비게이션 표시 | 카테고리별 탐색을 위한 기본 인터페이스 | 홈 페이지, 카테고리 페이지 |

## 📄 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F002`, `F004`, `F005`, `F006`, `F010`, `F012` | **메뉴 위치:** 헤더 "로고/홈" 클릭

| 항목 | 내용 |
|------|------|
| **역할** | 블로그의 입구로서 최근 글 목록과 카테고리 네비게이션을 제공하는 메인 랜딩 페이지 |
| **진입 경로** | 블로그 첫 방문 또는 헤더의 로고/홈 버튼 클릭 |
| **사용자 행동** | 최근 글을 카드 형태로 브라우징, 카테고리 버튼 클릭으로 필터링, 검색창에 키워드 입력 |
| **주요 기능** | • 최근 발행된 글 목록을 카드 형태로 표시 (제목, 썸네일, 카테고리, 발행일)<br>• 전체 카테고리 버튼/탭으로 실시간 필터링<br>• 검색창에서 제목 및 태그 검색<br>• 반응형 레이아웃 (모바일: 1칼럼, 태블릿: 2칼럼, 데스크톱: 3칼럼)<br>• 글 카드 클릭 시 상세 페이지로 이동 |
| **다음 이동** | 글 카드 클릭 → 글 상세 페이지, 카테고리 클릭 → 카테고리 페이지, 검색 엔터 → 글 목록 (검색 결과) |

## 🗄️ 데이터 모델

### Post (블로그 글)
블로그의 개별 글 정보로, Notion 데이터베이스에서 직접 조회됨

| 필드 | 설명 |
|------|------|
| id | Notion 페이지 고유 ID (notion_page_id) |
| title | 글 제목 |
| category | 카테고리 (예식장, 스드메, 예물, 하객 선물, 신혼살림, 허니문) |
| tags | 태그 배열 (다중 선택) |
| published_date | 발행일 (Notion date 필드) |
| status | 상태 (초안/발행됨) - 발행됨 상태만 블로그에 표시 |
| content | Notion 페이지 블록 콘텐츠 |

### Category (카테고리)
블로그 카테고리 정보

| 필드 | 설명 |
|------|------|
| name | 카테고리명 (예: 예식장, 스드메 등) |
| slug | 카테고리 슬러그 (정렬용) |
| description | 카테고리 설명 (옵션) |
