# 프로덕션 배포 스모크 테스트 체크리스트

이 문서는 Vercel 배포 후 프로덕션 환경에서 수행해야 할 스모크 테스트 체크리스트입니다.

## 📋 테스트 전 준비

- [ ] 배포가 완료되었으며 프로덕션 URL이 준비되어 있는가?
- [ ] 새 브라우저 탭에서 시크릿/프라이빗 모드로 테스트
- [ ] 네트워크 상태 확인 (온라인 상태 필수)
- [ ] 여러 기기(데스크톱, 태블릿, 모바일)에서 테스트

## 🧪 테스트 시나리오

### 1단계: 사이트 기본 접근성 (5분)

#### 1.1 홈페이지 접근

```
테스트: https://[your-domain]/ 접근
예상:
- 페이지가 로드되고 HTTPS 보안 연결 확인
- 로딩 시간이 3초 이내
- 모든 CSS/JavaScript 로드됨 (개발자 도구 네트워크 탭에서 에러 없음)
```

**검증 항목**

- [ ] 페이지 로드 성공 (상태 코드 200)
- [ ] HTTPS 보안 연결 (주소창에 자물쇠 아이콘)
- [ ] 로딩 시간 < 3초
- [ ] 콘솔에 에러 없음 (개발자 도구 F12 > Console 탭)

#### 1.2 네비게이션 렌더링

```
테스트: 홈페이지 상단 헤더 확인
예상:
- 블로그 제목/로고 표시
- 카테고리 네비게이션 메뉴 표시
- 검색창 표시
```

**검증 항목**

- [ ] 헤더 로고/제목 표시
- [ ] 카테고리 메뉴 버튼 표시 (예식장, 스드메, 예물 등)
- [ ] 검색창 표시 및 입력 가능
- [ ] 헤더 스타일 정상 (색상, 배치)

---

### 2단계: 콘텐츠 로딩 (5분)

#### 2.1 포스트 목록 로드

```
테스트: 홈페이지 포스트 목록 확인
예상:
- 최소 1개 이상의 포스트 카드 표시
- 포스트 제목, 이미지 썸네일, 메타데이터 표시
- 카드가 클릭 가능한 상태
```

**검증 항목**

- [ ] 포스트 카드 1개 이상 표시
- [ ] 각 카드에 제목 표시
- [ ] 각 카드에 카테고리 배지 표시
- [ ] 각 카드에 발행일 표시
- [ ] 포스트 이미지/썸네일 표시 (없으면 기본 이미지)
- [ ] 응답형 그리드 정상 작동 (모바일: 1열, 태블릿: 2열, 데스크톱: 3열)

#### 2.2 데이터 정확성

```
테스트: Notion에 존재하는 포스트가 표시되는지 확인
예상:
- 발행 상태의 포스트만 표시
- 발행일 순서대로 정렬 (최신순)
- 포스트 메타데이터 정확함 (카테고리, 태그, 날짜)
```

**검증 항목**

- [ ] 표시된 포스트가 Notion에 발행된 글과 일치
- [ ] 미발행 상태의 글은 표시되지 않음
- [ ] 포스트가 최신순으로 정렬됨
- [ ] 카테고리 분류 정확함
- [ ] 메타데이터 정보 정확함

---

### 3단계: 포스트 상세 페이지 (10분)

#### 3.1 포스트 상세 접근

```
테스트: 홈페이지에서 임의의 포스트 카드 클릭
예상:
- 포스트 상세 페이지로 이동
- URL이 /post/[id] 형태
- 포스트 제목, 본문, 메타데이터 표시
```

**검증 항목**

- [ ] 포스트 상세 페이지 로드 성공
- [ ] URL 형식 정상 (/post/로 시작)
- [ ] 포스트 제목 표시
- [ ] 포스트 본문 표시 (Notion 블록)
- [ ] 로딩 시간 < 2초

#### 3.2 Notion 블록 렌더링

```
테스트: 포스트 본문의 다양한 블록 타입 확인
예상:
- 텍스트 단락 정상 렌더링
- 제목 (H1, H2, H3) 정상 렌더링
- 이미지 정상 로드 및 표시
- 리스트 (글머리, 번호) 정상 렌더링
- 인용구 정상 렌더링
- 표 정상 렌더링
```

**검증 항목**

- [ ] 텍스트 본문 렌더링 정상
- [ ] 제목 스타일 정상 (크기, 색상)
- [ ] 이미지 로드 및 표시 정상
- [ ] 이미지 크기 반응형 대응
- [ ] 리스트 형식 정상
- [ ] 인용구 스타일 정상
- [ ] 표 렌더링 정상

#### 3.3 메타데이터 확인

```
테스트: 포스트 상세 페이지의 메타데이터 영역 확인
예상:
- 발행일 표시
- 카테고리 표시
- 태그 표시 (있는 경우)
- 메타데이터 영역 위치 (상단 또는 하단)
```

**검증 항목**

- [ ] 발행일 정확하게 표시
- [ ] 카테고리 정확하게 표시
- [ ] 태그 표시 (있는 경우)
- [ ] 메타데이터 스타일 정상

#### 3.4 네비게이션

```
테스트: 포스트 상세 페이지 내 네비게이션 기능
예상:
- "목록으로 돌아가기" 또는 "뒤로 가기" 버튼 작동
- 홈페이지로 이동 가능
```

**검증 항목**

- [ ] 돌아가기 버튼 표시
- [ ] 돌아가기 버튼 클릭 시 목록으로 이동
- [ ] 헤더 로고/제목 클릭 시 홈으로 이동

### 4단계: 카테고리 필터링 (10분)

#### 4.1 홈페이지 카테고리 버튼

```
테스트: 홈페이지 카테고리 필터 버튼 클릭
예상:
- 각 카테고리 버튼 표시
- 카테고리 선택 시 해당 카테고리 글만 표시
- 선택된 카테고리 버튼이 강조됨
```

**검증 항목**

- [ ] 모든 카테고리 버튼 표시 (예식장, 스드메, 예물, 하객 선물, 신혼살림, 허니문)
- [ ] 카테고리 버튼 클릭 시 해당 글 필터링
- [ ] 선택된 카테고리 버튼 강조 표시
- [ ] 필터링된 결과가 정확함

#### 4.2 카테고리 페이지 접근

```
테스트: /category/[slug] 경로로 직접 접근
예상:
- 카테고리 페이지 로드
- 카테고리 제목/설명 표시
- 카테고리별 글 목록 표시
```

**검증 항목**

- [ ] 카테고리 페이지 로드 성공
- [ ] 카테고리 제목 정확하게 표시
- [ ] 카테고리 설명 정확하게 표시
- [ ] 해당 카테고리의 글만 표시
- [ ] 필터링 결과 정확함

#### 4.3 카테고리 페이지에서 포스트 상세 접근

```
테스트: 카테고리 페이지에서 포스트 클릭
예상:
- 포스트 상세 페이지 로드
- 포스트 정보 정확함
```

**검증 항목**

- [ ] 포스트 상세 페이지 로드 성공
- [ ] 포스트 정보 정확함
- [ ] 포스트 상세 페이지에서 카테고리 페이지로 돌아가기 가능

#### 4.4 존재하지 않는 카테고리

```
테스트: /category/invalid-slug 또는 잘못된 카테고리로 접근
예상:
- 404 또는 빈 결과 페이지 표시
```

**검증 항목**

- [ ] 잘못된 카테고리 접근 시 적절한 에러/빈 상태 처리

### 5단계: 검색 기능 (10분)

#### 5.1 기본 검색

```
테스트: 홈페이지 검색창에서 검색어 입력
예상:
- 검색창에 텍스트 입력 가능
- Enter 키 또는 검색 버튼으로 검색 실행
- 검색 결과 페이지 로드 (/search?q=...)
```

**검증 항목**

- [ ] 검색창 입력 가능
- [ ] 검색어 전송 가능
- [ ] 검색 결과 페이지 로드

#### 5.2 검색 결과 정확성

```
테스트: 다양한 검색어로 검색 수행
예상:
- 제목에 일치하는 글 표시
- 태그에 일치하는 글 표시
- 일치도 높은 순으로 정렬
```

**검증 항목**

- [ ] 검색어 포함 포스트 정확하게 표시
- [ ] 관련 없는 포스트는 표시되지 않음
- [ ] 검색 결과 개수 정확함
- [ ] 검색 결과 정렬 정확함 (일치도 순)

#### 5.3 검색 결과 페이지

```
테스트: 검색 결과 페이지 UI 확인
예상:
- "N개의 검색 결과" 메시지 표시
- 검색된 포스트 카드 표시
- 검색 결과 포스트 클릭 시 상세 페이지 이동
```

**검증 항목**

- [ ] 검색 결과 개수 메시지 표시
- [ ] 포스트 카드 정상 표시
- [ ] 포스트 클릭 시 상세 페이지 로드

#### 5.4 검색 결과 없음

```
테스트: 일치하는 글이 없는 검색어로 검색
예상:
- "검색 결과가 없습니다" 메시지 표시
- 빈 결과 상태 UI
```

**검증 항목**

- [ ] 결과 없음 메시지 표시
- [ ] 친절한 안내 메시지 제공

#### 5.5 검색창 유지

```
테스트: 검색 후 홈페이지로 돌아가기
예상:
- 검색창 여전히 작동
- 새 검색 가능
```

**검증 항목**

- [ ] 검색창 계속 작동
- [ ] 여러 번 검색 가능

#### 6.1 404 페이지

```
테스트: 존재하지 않는 경로로 접근
예상:
- /nonexistent 또는 /invalid/path로 접근
- 404 에러 페이지 표시
- 홈으로 돌아가기 버튼 제공
```

**검증 항목**

- [ ] 존재하지 않는 경로 접근 시 404 페이지 표시
- [ ] 404 페이지 UI 정상 렌더링
- [ ] 홈으로 돌아가기 버튼 작동

#### 6.2 존재하지 않는 포스트

```
테스트: /post/invalid-id 또는 존재하지 않는 포스트 ID로 접근
예상:
- 404 또는 에러 페이지 표시
```

**검증 항목**

- [ ] 존재하지 않는 포스트 접근 시 에러 처리
- [ ] 적절한 에러 메시지 표시

### 7단계: SEO & 메타데이터 (5분)

#### 7.1 페이지 제목 및 설명

```
테스트: 각 페이지의 HTML 메타 태그 확인
방법: 개발자 도구 (F12) > Elements 탭 > <head> 섹션
예상:
- <title> 태그 적절함
- <meta name="description"> 태그 존재
- <meta property="og:*"> 태그 존재
```

**검증 항목 - 홈페이지**

- [ ] `<title>` 적절함 ("웨딩 비용 정리 블로그" 포함)
- [ ] `<meta name="description">` 존재
- [ ] `<meta property="og:title">` 존재
- [ ] `<meta property="og:description">` 존재

**검증 항목 - 포스트 상세 페이지**

- [ ] `<title>` = 포스트 제목
- [ ] `<meta name="description">` = 포스트 설명
- [ ] `<meta property="og:title">` = 포스트 제목
- [ ] `<meta property="og:image">` 존재 (포스트 이미지)

**검증 항목 - 카테고리 페이지**

- [ ] `<title>` = 카테고리 제목
- [ ] `<meta name="description">` 존재

#### 7.2 robots.txt & sitemap.xml

```
테스트: 검색 엔진 파일 접근
```

**검증 항목**

- [ ] https://[domain]/robots.txt 접근 가능
- [ ] robots.txt 내용 적절함 (Allow: /)
- [ ] https://[domain]/sitemap.xml 접근 가능
- [ ] sitemap.xml에 주요 경로 포함

#### 7.3 소셜 공유

```
테스트: 포스트 페이지를 소셜 미디어(Twitter, Facebook 등)에서 공유 시뮬레이션
방법: 각 소셜 미디어의 URL 미리보기 도구 사용
예상:
- OG 태그를 기반으로 미리보기 생성
- 제목, 설명, 이미지 정상 표시
```

**검증 항목**

- [ ] Facebook 미리보기 정상 (og:title, og:description, og:image)
- [ ] Twitter 미리보기 정상
- [ ] LinkedIn 미리보기 정상

### 8단계: 반응형 디자인 (10분)

#### 8.1 모바일 (375px)

```
테스트: 개발자 도구 Device Emulation에서 iPhone SE (375px) 선택

**검증 항목**

- [ ] 수평 스크롤 없음
- [ ] 카테고리 필터 버튼 스크롤 가능
- [ ] 포스트 카드 정상 렌더링
- [ ] 포스트 상세 페이지 반응형 정상
- [ ] 검색 결과 정상

#### 8.2 태블릿 (768px)

```
테스트: 개발자 도구 Device Emulation에서 iPad (768px) 선택
```

**검증 항목**

- [ ] 홈페이지 반응형 정상 (2열 그리드)
- [ ] 포스트 카드 2열 레이아웃
- [ ] 모든 요소 적절한 여백
- [ ] 수평 스크롤 없음

#### 8.3 데스크톱 (1024px+)

```
테스트: 개발자 도구 Device Emulation에서 데스크톱 모드 (1440px) 선택

- [ ] 홈페이지 반응형 정상 (3열 그리드)

- [ ] 전체 콘텐츠 한 화면에 보임

#### 8.4 폰트 및 이미지 스케일링

```
테스트: 브라우저 줌 레벨 조정
```

**검증 항목**

- [ ] 200% 줌 시 레이아웃 깨지지 않음
- [ ] 이미지 비율 유지
- [ ] 텍스트 overflow 없음

### 9단계: 성능 (5분)

#### 9.1 Lighthouse 점수 측정

```
테스트: Google Lighthouse 사용
방법:
1. Chrome 개발자 도구 (F12) 열기
2. Lighthouse 탭 클릭
3. "Analyze page load" 클릭
4. 각 점수 확인
```

**검증 항목**

- [ ] Performance >= 80
- [ ] Accessibility >= 90
- [ ] Best Practices >= 85
- [ ] SEO >= 90
